Viewing alarms¶

In the ** Alarm ** tab menu, you can check the alarm history that has occurred so far. Unlike the: ref: Statistics page, which shows the overall status of the alarm, this menu provides an optimized UI for viewing and browsing more individual alarms.

This tab consists of the following two pages.

  • Alarm List

  • Alarm Details

Alarm List¶

When entering the Alarm tab, the alarms that have occurred so far are listed and displayed.Using the Alarm rule / Timeline selection box at the top of the screen, you can sort the alarm list by alarm rule or by the time that occurred.

  • Alarm rule (sort by alarm rule)

  • Timeline (sort by occurrence time)

Click + Load moreat the end of a category to show more alarm entries in that category.

Alarm Details¶

Select one of the items listed in the alarm list to view detailed information about the alarm. Below is the description of each area of the alarm detail page.

Summary¶

This area shows the occurrence status of the alarm. If an alarm occurs continuously according to a specified period, it is maintained as one alarm item, and the number of data points exceeding the severity level of the alarm is also displayed. After confirming, it provides a link to record the result of the processing. If it is not the alarm rule that I created, you can receive the alarm by pressing Subscribe.

The example below shows that the alarm occurred twice in a row (Alarms), and because the alarm check interval is 1 minute, two alarms lasted for a total of 120 seconds. (Elapsed Time).

Alarm History¶

This area shows the history of alarms generated by alarm rules applied to the alarm.

Alarm Confirmation¶

The first menu on the right tab provides a communication function to check the alarm subscriber list after checking the alarm (Subscriber), check the alarm to record the status (Alarm Confirm), and leave a worker record.

There are four types of alarm confirmation items.

  • Not Processed: No action has been taken on the alarm

  • Stop notification: The user acknowledges the alarm and no longer receives notification

  • Processing completed: After checking the alarm, the action is completed. Alarm related tags can be added

  • False Alarm: An alarm that occurs when the condition is not abnormal

Subscribers can search and add users related to the alarm by ID, and can set the alarm to be sent by e-mail if necessary.

Alarm Rule¶

This area shows the severity of the alarm, the time the alarm occurred, and the settings of the rule that triggered the alarm and information related to the data source.

Chart View¶

This tab area shows the trend of abnormal scores of the aggregated data monitored within the corresponding alarm section.The number of occurrences according to the alarm severity (Critical, Major, Moderate, Low) is also shown at the top of the chart. Please refer to: ref: basic_principles for how the chart is calculated.

  • Total abnormal score: Shows abnormal score for all measure columns included in the alarm rule.

  • Chart by measures: Shows the trend between the predicted value and the actual value of each individual measure column data included in the alarm rule.

Table View¶

This area shows the actual and predicted data and abnormal score for each alarm occurrence.
