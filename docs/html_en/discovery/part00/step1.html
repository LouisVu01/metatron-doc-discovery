Step 1. Creating a data source¶

The first step in data analysis is loading your data to the system. Metatron Discovery allows you to easily load data of various sources.

The example in this tutorial shows you how to load data stored in your local directory. First, prepare the data. An Excel file (.xls, .xlsx) or .csv file will suffice. The sample file involves sales data. Download it from the link below.

sample data (.csv)

Data source can be viewed and loaded from Management > Data Storage > Data Source. To create a new data source, click the New button on the upper right of the data source list.

In this tutorial, click File to retrieve the data from your local directory. See Creating Data Sources for details on creating a data source from other sources.

Drag and drop the data you wish to analyze, or retrieve it from the directory.

Drag your cursor over the sales data to view up to 100 rows of data with detection of column and line separators. This data is properly displayed using the default separators. Click Next.

To view the data, the column type should be adjusted accordingly. This task is called Data Schema Construction.

Each column plays the role of “dimension” or “measure.” See &lsquo;Dimensions&rsquo; and &lsquo;Measures&rsquo; for further details. In this data, the Discount, Profit, Quantity, Sales, DaystoShipActual, SalesForecast, DaystoShipScheduled, SalesperCustomer, and ProfitRatio columns must be converted into measurements.

Next, the data type of columns must be adjusted accordingly. Characters are the default setting for “dimension,” and integers for “measure.” Change the settings so that the data type best matches the sample. Below is a list of items to be modified in this data.

  • Orderdate : Date/time

  • Discount : Decimal

  • ShipDate : Date/time (Change time format to yyyy. MM. dd. and click the checkbox to validate)

  • SalesperCustomer : Decimal

  • ProfitRatio : Decimal

  • latitude : Latitude

  • longitude : Longitude

Lastly, you should create a new column. Since we have columns for latitude and longitude, we can create a point type column. Click the Add column button on the upper right. Select the latitude column for the Latitude column, and the longitude column for the Longitude column. Name the columns appropriately, and click Add. A new point type column is created!

Once you are done with schema construction, click Next. If necessary, you can change the collection settings of Druid. The default settings are sufficient for this tutorial.

Lastly, enter the Name and Description for the data source. Click Done to proceed to the details page.

In the details page, you can view the loading status in real time. The screen below appears after a few minutes, indicating success. A histogram is displayed. If you encounter an error while loading another data source, click Details to view the Druid loading log. Loading may be unsuccessful due to repeated column names or mismatch between data and column types. Try loading again after addressing the issue.

To make the data source available to other users, check the box next to Permit use of data source in all workspaces under Assignment. To make the data source available only to specific users, click Modify and select individual users or team workspaces as desired.

In this example, we will choose Open Data to make it available to all users.

The loaded data can be viewed under the Data tab.

Congratulations! Now, it’s time to use the data source. Let’s proceed to the next step.
